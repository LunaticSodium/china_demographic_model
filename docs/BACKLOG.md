### 渲染像素对齐问题

- 当前 PyramidView 用比例计算 `rowH = h / (maxAge + 1)`（h = canvas 实际高度）。
- 最大化窗口后，rowH 是浮点数（如 9.83333…），各行被四舍五入到不同像素 → 间距像素不一致。
- 修法：**对像素对齐要求高的数值，不应该用画面大小比例算，而是用固定值；固定值再挂钩窗口大小**。
- 具体方案：
  - 选个基础逻辑像素（如 8px/行）
  - 整体 Canvas 内容高度 = 基础值 × 行数 = 808px 定值
  - 用 Viewbox / LayoutTransform 把整体按窗口大小整数倍缩放
  - 或：每行 rowH = `Math.Floor(h / (maxAge + 1))` + 累加 fractional 部分到指定行
- 类似的问题可能也存在于 YearSlider 的年份刻度（每年宽度 = w / years 也是浮点）。
